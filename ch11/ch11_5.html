11.5 Let束縛

MLでは\(\mathrm{let} \ \mathrm{x} = \mathrm{t}_1 \ \mathrm{in} \ \mathrm{t}_2\)と書くと、 「式\(\mathrm{t}_1\)を評価し、その結果の値に名前\(\mathrm{x}\)を束縛した状態で\(\mathrm{t}_2\)を評価せよ」 という意味になる。

新しい構文形式

\[ \begin{align} t \ ::= \ & ... \\ & \mathrm{let} \ \mathrm{x}=\mathrm{t} \ \mathrm{in} \ \mathrm{t} \tag{let束縛} \end{align} \]

新しい評価規則

\[ \mathrm{let} \ \mathrm{x}=\mathrm{v}_1 \ \mathrm{in} \ \mathrm{t}_2 \to [\mathrm{x} \mapsto \mathrm{v}_1]\mathrm{t}_2 \tag{E-LetV} \]

新しい型付け規則

\[ \frac {\mathrm{t}_1 \to \mathrm{t}_1'} { \mathrm{let} \ \mathrm{x}=\mathrm{t}_1 \ \mathrm{in} \ \mathrm{t}_2 \to \mathrm{let} \ \mathrm{x}=\mathrm{t}_1' \ \mathrm{in} \ \mathrm{t}_2 } \tag{E-Let} \]

演習11.5.1

型検査器letexerciseは、let式の不完全な実装である。基本的な構文解析と表示の関数は実装されているが、 eval1関数およびtypeof関数はTmLetのための節が欠けている。これを完成させよ。

新しい派生形式

\[ \mathrm{let} \ \mathrm{x}=\mathrm{t}_1 \ \mathrm{in} \ \mathrm{t}_2 \stackrel{\text{def}}{=} (\lambda \mathrm{x} : \mathrm{T}_1 . \mathrm{t}_2) \mathrm{t}_1 \]

演習11.5.2

letを派生形式として定義する別の方法として、直ちに「実行」する方法、すなわち \(\mathrm{let} \ \mathrm{x}=\mathrm{t}_1 \ \mathrm{in} \ \mathrm{t}_2\)を、 本体を置換した\([\mathrm{x} \mapsto \mathrm{t}_1]\mathrm{t}_2\)の略記とみなす方法が考えられる。 これは良いアイデアだろうか？

ダメ。

  1. 評価順が変わってしまう。
  2. \(\mathrm{t}_1\)が正しく型付けできないことを保証できなくなる。
